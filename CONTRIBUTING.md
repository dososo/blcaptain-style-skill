# Contributing

## Design contribution rules

A new style is accepted only if it defines:

- typography hierarchy
- palette tokens
- density rule
- image behavior
- when to use
- when not to use

A new layout is accepted only if it defines:

- layout purpose
- fixed structure
- required fields
- optional fields
- mobile readability risk
- validation risks

## Code contribution rules

- Keep the CLI runnable.
- Do not introduce a full frontend build chain unless necessary.
- Prefer semantic HTML.
- Every poster must be independently renderable.
- Keep image provenance tracking intact.

## Visual quality bar

A PR should include at least one generated HTML demo and one rendered PNG sample when possible.
