# 00 — 方法研究

本文记录我们从优秀的开源社交卡片项目与业界最佳实践中学习的方法层内容。

核心边界：只学习产品化方法，不复制视觉风格、模板、代码、主题名、布局细节或资产。

## 参考来源

- 优秀开源社交卡片项目的公开 README / PRODUCT.md / SKILL.md 等文档
- 业界获奖编辑设计、杂志排版与图文卡片的公开案例

## 关键学习

### 1. Skill 是产品，不是提示词

值得学习的是结构化生产方式：

- 明确目标用户。
- 明确适合 / 不适合场景。
- 明确视觉系统。
- 明确固定输出尺寸。
- 明确布局配方。
- 明确图片工作流。
- 明确验证规则。
- 明确反模式。

BLCaptain 对应动作：先稳定产品文档、合同和 gate，再扩样张。

### 2. 质量来自流程约束

社交卡不是“快速生成图片”就够。核心目标是让输出不像松散 AI 模板，而是可读、可信、可复用。

BLCaptain 对应动作：技术 PASS 不等于视觉 PASS；人工视觉确认必须保留。

### 3. 两套视觉系统已经足够

当前只保留：

- Still Paper。
- Signal Proof。

不得新增第三套主视觉系统。`SL-05 Signal Noir` 保持候选状态，未补视觉方向稿前不实现。

### 4. 种子模板是隐形设计师

固定版式工作流必须先固定：

- token。
- 间距。
- 字阶。
- 布局区块。
- 图片规则。
- 失败规则。

BLCaptain 对应动作：Agent 不得每次从零发明 CSS。

### 5. 硬限制是产品承诺

每个布局合同都需要：

- 标题字数上限。
- 最大行数。
- 正文长度限制。
- 页脚限制。
- 图片比例。
- 密度规则。
- 失败状态。

### 6. 图片来源记录是工作流，不是装饰

每张 image-led 卡必须定义：

- 图片角色。
- 来源。
- `object-position`。
- 文本安全区。
- 遮罩规则。
- 成图内是否标来源。

所有外部图片必须写入 `SOURCES.md`。

### 7. Validator 是护栏，不是审美裁判

机器验证负责：

- overflow。
- 字数和行数限制。
- footer collision。
- image frame overflow。
- style identity。
- 图片来源记录。

人工视觉确认负责：

- 审美。
- 氛围。
- 图片质量。
- 是否惊艳且好用。
- 是否值得发布。

## 不得复制

- 视觉构图。
- 精确布局配方。
- 主题名。
- 配色值。
- 字体组合。
- 边注样式。
- 卡片细节。
- 截图资产。
- 实现代码。
- README / SKILL 文案。

## 可以借鉴

- 七步工作流。
- 固定种子模板思路。
- 主题限制原则。
- 布局配方概念。
- 图片来源记录。
- 验证分类。
- 反模式文档。
- 产品边界意识。

学习生产纪律。

建立自己的 Still Paper / Signal Proof 视觉语言。
