# 标题压缩规则

长标题进入卡片前必须先压缩，不能靠缩小字号硬塞。当前 `SP-C01 · Still Paper · Mist Field` 仍是 `PENDING_USER_REVIEW`，标题调整也要进入人工视觉判断。

## 适用场景

- 小红书封面。
- 公众号封面。
- Still Paper 长文拆卡。
- Signal Proof 截图证据卡。

当前不得进入 SL-C02；这里只记录后续标题规则。

## 压缩步骤

1. 去掉冒号、破折号后的副标题，必要时改成 subtitle。
2. 保留最强动词、冲突或结果。
3. 删除填充词：`一个`、`这件事`、`其实`、`关于`、`如何看待`。
4. 去掉抽象大词，改成具体对象或动作。
5. 仍然过长时，拆成 `title + subtitle`，不要缩小字号。

## 长度目标

```text
SP-C01 Clean Hero: 8-16 个中文字符
Still Paper 普通封面: 8-18 个中文字符
Signal Proof 证明卡: 8-16 个中文字符
小红书封面: 6-18 个中文字符
公众号封面: 12-22 个中文字符
```

## 好模式

- 别再做 A，先解决 B
- X 不是 Y，是 Z
- 一张图看懂 X
- X 的真正瓶颈
- 先看证据，再下判断
- 这一步决定成败

## Still Paper 标题

- 可以更安静，但必须有判断。
- 不写空泛诗句。
- 标题和图片要互相加分。

## Signal Proof 标题

- 要指向证据或结果。
- 不写空泛效率口号。
- 截图卡标题不能压住关键 UI。

## 失败条件

- 缩字号解决超长标题。
- 标题超过合同行数。
- 标题像课程目录。
- 标题与图片主体无关。
- 技术 PASS 后跳过 `manual:review`。

发布前仍以 `release:ready` 为准。
